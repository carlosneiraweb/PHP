px_get_info

(PECL paradox >= 1.0.0)

px_get_info — Devuelve una gran cantidad de información sobre un archivo paradox

Descripción

array px_get_info ( resource $pxdoc )

Parámetros

pxdoc

Identificador de recursos de la base de datos paradox devuelto por la función px_new().

Valores devueltos

Devuelve una matriz asociada con una gran cantidad de información sobre un archivo paradox. Esta matriz es probable que sea ampliada en el futuro.

fileversion

Versión del archivo multiplicado por 10, por ejemplo, 70.

tablename

Nombre de la tabla tal como fué almacenada en el archivo. Si la base de datos fue creada con pxlib, entonces este será el nombre del archivo sin la extensión.

numrecords

Número de registros en esta tabla.

numfields

Número de campos en esta tabla.

headersize

Número de bytes utilizados para la cabecera. Esto es por lo general 0x800.

recordsize

Número de bytes utilizados por cada registro. Esta es la suma de todos los tamaños de campo (disponible desde la versión 1.4.2).

maxtablesize

Este valor, multiplicado por 0x400 es el tamaño de un bloque de datos en bytes. El número máximo de registros en un bloque de datos es la parte entera de (maxtablesize * 0x400 - 8) / recordsize.

numdatablocks

El número de bloques de datos en el archivo. Cada bloque de datos contiene un cierto número de registros que dependen del tamaño del registro y del tamaño del bloque de datos (maxtablesize). Los bloques de datos pueden no necesariamente estar rellenados por completo.

numindexfields

Número de campos utilizados para el índice primario. Los campos siempre se inician con el campo número 1.

codepage

El sistema de codificación DOS que fue utilizado para la codificación de los campos con datos de caracteres. Si la codificación de destino no se ha establecido con la función px_set_targetencoding() esta será la codificación de los campos de caracteres cuando se accede a los registros con la función px_get_record() o px_retrieve_record().

  • px_numrecords() - Devuelve el número de registros en una base de datos
