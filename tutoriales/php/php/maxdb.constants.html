Constantes predefinidas

Estas constantes están definidas por esta extensión y estarán disponibles sólo cuando la extensión haya sido compilada con PHP, o bien sea cargada dinámicamente en ejecución.

Las siguientes constantes están definidas para usarse con maxdb_options(). Para una descripción más detallada de estas constantes, véase » http://maxdb.sap.com/documentation/.

MaxDB PHP client constants
Constante Descripción
MAXDB_COMPNAME El nombre del componente usado para inicializar el entorno en tiempo de ejecución SQLDBC.
MAXDB_APPLICATION La aplicación a conectar a la base de datos.
MAXDB_APPVERSION La versión de la aplicación.
MAXDB_SQLMODE El modo de SQL.
MAXDB_UNICODE TRUE, si la conexión es un cliente unicode (UCS2) o FALSE si no.
MAXDB_TIMEOUT El tiempo de inactividad máximo permitido después del cual la conexión a la base de datos es cerrada por el sistema.
MAXDB_ISOLATIONLEVEL Especifica si los bloqueos compartidos y exclusivos son implicitamente a petición o liberados y cómo.
MAXDB_PACKETCOUNT El número de paquetes diferentes de peticiones usados para la conexión.
MAXDB_STATEMENTCACHESIZE El número de sentencias preparadas a ser colocadas en memoria para la conexión para ser reusadas.
MAXDB_CURSORPREFIX El prefijo a usar para las tablas de resultados que son nominadas automáticamente.

La función maxdb_fetch_array() utiliza una constante para los diferentes tipos de arrays de resultados. Están definidas las siguientes constantes:

Constantesde obtención de MaxDB
Constante Descripción
MAXDB_ASSOC Las columnas son devueltas en un array teniendo como índices el nombre del campo.
MAXDB_ASSOC_UPPER Las columnas son devueltas en un array teniendo como índices el nombre en mayúsculas del campo.
MAXDB_ASSOC_LOWER Las columnas son devueltas en un array teniendo como índices el nombre en minúsculas del campo.
MAXDB_BOTH Las columnas son devueltas en un array teniendo como índices un índice numérico y el nombre del campo.
MAXDB_NUM Las columnas son devueltas en un array teniendo como índices un índice numérico de los campos. Este índice comienza con 0, el primer campo del resultado.
