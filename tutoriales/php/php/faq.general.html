¿Qué es PHP?

Según el prefacio del manual:

PHP es un lenguaje de «scripting» que puede ser embebido en HTML. Gran parte de su sintaxis se toma prestada de C, Java y Perl con un par de características específicas propias de PHP. El objetivo del lenguaje es permitir a los desarrolladores web escribir con rapidez páginas generadas dinámicamente.

¿Qué significa PHP?

PHP significa PHP: Hypertext Preprocessor. Esto confunde a mucha gente porque la primera palabra del acrónimo es el acrónimo mismo. Este tipo de acrónimo se denomina 'acrónimo recursivo'. Para más información, los curiosos pueden visitar » El Diccionario de Informática On-Line Gratuito ó la » entrada de acrónimo recursivo en Wikipedia (ambos en inglés).

¿Cuál es la relación entre las versiones?

PHP/FI 2.0 es una versión de PHP antigua y sin soporte. PHP 3 es el sucesor de PHP/FI 2.0 y es mucho mejor. PHP 5 es la generación actual de PHP, la cual emplea el » Motor Zend 2 que, entre otras cosas, ofrece muchas características adicionales de POO.

¿Puedo usar varias versiones de PHP al mismo tiempo?

Sí. Vea el fichero INSTALL que se incluye en la distribución del código fuente de PHP.

¿Cuáles son las diferencias entre PHP 4 y PHP 5?

A pesar de que PHP 5 fue diseñado con la intención de ser lo más compatible posible con versiones anteriores, existen algunos cambios significativos. Algunos de estos son:

  • Un nuevo modelo de POO basado en el Motor Zend 2.0
  • Una nueva extensión que mejora el soporte para MySQL
  • Soporte nativo integrado para SQLite
  • Una nueva constante de notificación de errores, E_STRICT, para sugerencias de código en tiempo de ejecución.
  • Una multitud de nuevas funciones para simplificar la autoría de código (y reducir la necesidad de escribir funciones propias para procedimientos comunes)
Para información más detallada, por favor vea la sección Migrar de PHP 4 a PHP 5 y la sección Cambios retroncompatibles.

¡Creo que he encontrado un error! ¿A quién debo informar?

Debería ir a la Base de datos de errores de PHP y asegurarse de que no se trate de un error ya descubierto. Si no lo encuentra en la Base de datos, use el formulario de notificación para informar del error. Es importante que utilice la Base de datos de errores en vez de solamente enviar un correo electrónico a una de las listas de correo, ya que así al error se le asignará un número de seguimiento, pudiendo volver luego y comprobar el estado de dicho error. La Base de datos de errores de PHP se encuentra en » http://bugs.php.net/.
