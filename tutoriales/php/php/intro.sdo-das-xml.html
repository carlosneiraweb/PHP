Introducción

Para poder usar el Servicio de Acceso a Datos XML de los Objetos de Datos de Servicio (SDO), se necesitará comprender algunos de los conceptos de SDO: el grafo de datos, el objeto de datos, XPath expresiones de propiedad, etc. Si no está familiarizado con estas ideas, debería ver primero la sección sobre SDO.

El trabajo del DAS XML es mover datos entre la aplicación y el origen de datos XML, el cual puede ser un fichero o un URL. Los SDOs siempre son creados y mantenidos según un modelo que define no nombres de los tipos y qué nombres de propiedad puede tener cada tipo. Para datos que provienen de XML, este modelo SDO se construye desde un fichero esquema escrito en el lenguaje de esquema XML (un fichero xsd). Este fichero de esquema se pasa normalmente al método create cuando se inicializa el DAS XML. La » especificación de SDO 2.0 define la referenciación entre los tipos XML y los tipos SDO. Existen varias pequeñas limitaciones en el soporte de PHP - no todo lo que está en la especificación puede hacerse - y estas limitaciones están resumidas en una sección posterior.
