SwishSearch::setLimit

(PECL swish >= 0.1.0)

SwishSearch::setLimit — Establecer los límites de la búsqueda

Descripción

void SwishSearch::setLimit ( string $property , string $low , string $high )
Advertencia

Esta función ha sido declarada EXPERIMENTAL. Su funcionamiento, nombre y la documentación que le acompaña puede cambiar sin previo aviso en futuras versiones de PHP. Utilícela bajo su propia responsabilidad.

Parámetros

property

Resultado búsqueda nombre propiedad.

low

El menor valor de la propiedad.

high

El mayor valor de la propiedad.

Valores devueltos

No devuelve ningún valor.

Errores/Excepciones

Lanza SwishException en caso de error.

Ejemplos

Ejemplo #1 Ejemplo básico de SwishSearch::setLimit()

<?php
try {

    $swish = new Swish("index.swish-e");
    $search = $swish->prepare();

    $results = $search->execute("time");
    echo "First query found: ", $results->hits, " hits\n";

    $i = 0;
    while($result = $results->nextResult()) {
        echo "Hit #", ++$i, " - ", $result->swishdocsize, " bytes\n";
    }

    $search->setLimit("swishdocsize", "3000", "6000"); //límite de tamaño del documento, desde 3000 hasta 6000 bytes
    $results = $search->execute("time");
    echo "Second query found: ", $results->hits, " hits\n";

    $i = 0;
    while($result = $results->nextResult()) {
        echo "Hit #", ++$i, " - ", $result->swishdocsize, " bytes\n";
    }

} catch (SwishException $e) {
    echo $e->getMessage(), "\n";
}

?>

El resultado del ejemplo sería algo similar a:

Hit #2 - 37937 bytes
Hit #3 - 7126 bytes
Hit #4 - 15427 bytes
Hit #5 - 4768 bytes
Second query found: 2 hits
Hit #1 - 4261 bytes
Hit #2 - 4768 bytes
