Otros cambios en las extensiones

Las siguientes extensiones ya no se pueden desactivar al generar la configuración:

  • PCRE
  • Reflection
  • SPL

Cambios en el comportamiento de extensiones, y nuevas funcionalidades:

  • Fecha y Hora - Ya no se usa la variable de entorno TZ para extraer la zona horaria
  • cURL - Ahora cURL soporta SSH
  • Network - dns_check_record() devuelve ahora un índice extra de "entradas", que contiene los elementos TXT.
  • Hash - Ahora se soportan los algoritmos hash SHA-224 y salsa.
  • mbstring - Ahora se soporta la codificación CP850.
  • OCI8 - Ahora, una llamada a oci_close() en una conexión persistente, o una variable referenciando a una conexión persistente fuera de ámbito, deshará todas las transacciones no validadas. Para evitar comportamientos inesperados, realice una validación explícita o deshaga según sus necesidad. Puede activar el comportamiento antiguo con la directiva INI oci8.old_oci_close_semantics. Ahora hay soporte para la Pila de Conexiones Residentes de Bases de datos (DRCP) y para Notificaciones de Aplicación Rápidas (FAN). Ahora hay soporte para Autenticación Externa en Oracle (salvo en Windows). La función oci_bind_by_name() ahora tiene soporte para SQLT_AFC (es decir, el tipo de datos CHAR).
  • OpenSSL - Ahora hay soporte para las funciones digest y cipher de OpenSSL. También es ahora posible acceder a los valores internos de las claves DSA, RSA y DH.
  • Manejo de Sesiones - Las sesiones ya no almacenan sus ficheros en "/tmp" cuando se aplique la restricción open_basedir, a no ser que se añada explícitamente "/tmp" a la lista de rutas permitidas.
  • SOAP Ahora permite enviar cabeceras HTTP facilitadas por el usuario.
  • MySQLi Ahora tiene soporte para conexiones persistentes, anteponiendo al nombre de host el prefijo "p:".
  • Procesado de Imágenes y GD El índice de "JPG Support" devuelto por gd_info() ha sido renombrado a "JPEG Support".
