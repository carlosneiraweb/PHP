Agrupamiento de conexiones

La agrupación de conexiones del complemento se crea cuando PHP inicializa sus módulos (MINIT), y se libera cuando PHP cierra los módulos (MSHUTDOWN). Lo mismo sucede con las conexiones persistentes de MySQL.

Según el modelo de desarrollo, la agrupación se usa durante la vida de una o múltiples peticiones web. Las conexiones de red están limitadas al tiempo de vida de un proceso al nivel del sistema operativo. Si el proceso de PHP sirve múltiples peticiones web, como es el caso de Fast-CGI o despliegues de servidor web con hilos, las conexiones de la agrupación pueden ser reutilizadas sobre conexiones múltiples. Ya que la multiplexación significa compartir conexiones, podría incluso acontecer con un despliegue de subprocesos donde dos hilos o dos peticiones web distintas estén vinculadas a una conexión de red de la agrupación.

Una conexión de la agrupación se cierra explícitamente una vez que la última referencia a ella sea liberada. Un cierre implícito ocurre cuando PHP cierra sus módulos.
