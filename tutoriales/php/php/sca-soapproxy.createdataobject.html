SCA_SoapProxy::createDataObject

(PECL SDO >= 0.5.0)

SCA_SoapProxy::createDataObject — Crear un SDO

Descripción

SDO_DataObject SCA_SoapProxy::createDataObject ( string $type_namespace_uri , string $type_name )
Advertencia

Esta función ha sido declarada EXPERIMENTAL. Su funcionamiento, nombre y la documentación que le acompaña puede cambiar sin previo aviso en futuras versiones de PHP. Utilícela bajo su propia responsabilidad.

Este método se usa dentro de un script de PHP ordinario o un componente SCA que necesita crear un SDO para pasarlo a un servicio web. Los parámetros son la URI del espacio de nombres del SDO deseado deseada y el nombre del tipo. El espacio de nombres y el tipo deben estar definidos en la interfaz del componente que va a ser invocado, por lo que el espacio de nombres y el tipo deben estar definidos dentro del WSDL para el servicio web. Si el servicio web también es un componente SCA, los tipos habrán sido definidos dentro de uno de los ficheros de esquemas que están especificados en la anotación @types dentro del componente para el cual el objeto SCA_SoapProxy es un delegado.

Parámetros

type_namespace_uri

El espacio de nombres del tipo.

type_name

El nombre del tipo.

Valores devueltos

Devuelve el recién creado SDO_DataObject.

Errores/Excepciones

SDO_TypeNotFoundException

Lanzada si namespaceURI y typeName no se corresponden con un tipo del WSDL que se usó para inicializar este SCA_SoapProxy.
