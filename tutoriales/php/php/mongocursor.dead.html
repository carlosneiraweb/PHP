MongoCursor::dead

(PECL mongo >=0.9.6)

MongoCursor::dead — Comprueba si hay resultados que no han sido enviados aún de la base de datos

Descripción

public bool MongoCursor::dead ( void )

La base de datos envía respuestas en lotes de documentos, de hasta 4 MB de documentos por repuesta. Este método comprueba si la base de datos tiene más lotes o si el conjunto de resultados se ha agotado.

Un cursor muerto ("dead") no significa que MongoCursor::hasNext() devuelva FALSE, solamente que la base de datos ha terminado de enviar resultados al cliente. Éste debería continuar iterando a través de los resultados hasta que MongoCursor::hasNext() sea FALSE.

Parámetros

Esta función no tiene parámetros.

Valores devueltos

Devuelve TRUE si hay más resultados que no han sido enviados aún al cliente, o FALSE de lo contrario.
